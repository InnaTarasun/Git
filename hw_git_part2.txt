GitHub. HW_2
1. На локальном репозитории сделать ветки для: git checkout -b 'имя ветки'
- Postman
- Jmeter
- CheckLists
- Bag Reports
- SQL
- Charles
- Mobile testing

2. Запушить все ветки на внешний репозиторий: git push origin 'имя ветки'
3. В ветке Bag Reports сделать текстовый документ со структурой баг репорта: vim structure_br.txt
4. Запушить структуру багрепорта на внешний репозиторий: git add structure_br.txt, git commit -m 'new file', git push
5. Вмержить ветку Bag Reports в Main: git checkout main, git merge Bag_reports
6. Запушить main на внешний репозиторий: git push
7. В ветке CheckLists набросать структуру чек листа: git checkout CheckLists, vim structure_cl.txt
8. Запушить структуру на внешний репозиторий: git add structure_cl.txt, git commit -m 'new file checklist', git push
9. На внешнем репозитории сделать Pull Request ветки CheckLists в main
10. Синхронизировать Внешнюю и Локальную ветки Main: git pull
